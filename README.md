# LifeOS

AstrBot 插件，个人写作与阅读数据追踪系统。通过 QQ 聊天即可记录和查询日常读写活动。

## 前置要求

- AstrBot >= v4.5.0
- SQLite >= 3.42

## 安装

将本目录放置到 AstrBot 插件目录下，重启机器人即可加载。

## 数据存储

| 存储 | 位置 | 用途 |
|------|------|------|
| SQLite 数据库 | `/data/lifeos/Database/lifeos.db` | 结构化记录，用于统计查询 |
| Markdown 日志 | `/data/lifeos/Data/YYYY-MM-DD.md` | 完整日志留存，用于回溯 |
| 规则文件 | `/data/lifeos/Config/` | 记录/查询的 LLM 规则模板 |

### 数据表

`writing_records` 和 `reading_records` 两张表结构相同，字段语义不同：

| 字段 | writing_records | reading_records |
|------|-----------------|-----------------|
| duration | 写作时长（小时） | 阅读时长（小时） |
| output_count | 产出字数（个） | 阅读章节数（章） |
| record_type | 正文 / 随笔 / 设计 / 设定 / 大纲 | 精读 / 随读 |

## 命令

### `/记录` — 记录活动

发送一条自然语言描述，插件自动解析并存入数据库。

```
/记录 写了《嘉豪》正文 1小时，1800字
/记录 读了《蛊真人》精读 2小时 12章
/记录 写《修仙界头号冤种》设定 500字 30分钟
/记录 今天重新部署了 AstrBot
```

支持的关键词：写了/码了/码字/写作/在写/完成了/更了/写、读了/看了/阅读/在读/在看/翻看/读

**处理流程**：
1. 本地完美匹配（无歧义时瞬间返回）→ 直接写入
2. 有歧义（多类型关键词、相似作品名等）→ 交给 LLM 解析
3. LLM 不可用时降级为本地解析兜底

### `/查询` — 查询统计

```
/查询          # 默认展示今日/本周/本月
/查询 今天     # 今日详情
/查询 本周     # 本周 TOP 3
/查询 本月     # 本月 TOP 3
/查询 近7天    # 近 7 天 TOP 3
/查询 近30天   # 近 30 天 TOP 3
```

复杂查询（如"对比本月和上月写作字数"）走 LLM 意图解析路径。

### `/helloworld` / `/test` — 测试命令

验证插件是否正常运行。

## 架构

```
用户消息 → 命令路由
  ├── /记录 → 活动拆分 → 本地完美匹配 / LLM解析 → DB写入 → 自然语言回复
  └── /查询 → 白名单本地查询 / LLM意图解析 → SQL执行 → 格式化输出
```

所有逻辑集中在 `main.py`（约 1400 行），不拆分子模块。

## 配置

通过 AstrBot 插件配置设置以下路径（均为可选，有默认值）：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| data_path | /data/lifeos/Data | Markdown 日志目录 |
| config_path | /data/lifeos/Config | 规则文件目录 |
| db_path | /data/lifeos/Database/lifeos.db | SQLite 数据库路径 |
